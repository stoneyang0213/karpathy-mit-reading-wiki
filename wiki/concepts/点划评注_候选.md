# 点划评注(候选)

> **来源**: 王荣生《阅读教学教什么》
> **Confidence**: VERIFIED(2026-04-14 stoneyang raw 原文采样)
> **状态**: _候选,目前只有 1 本书明确使用(王荣生原创)

## 王荣生原文(打靶组 C 直接命中)

> "王荣生把阅读方法概括为'**点划评注**':
> - **点和划**——眼睛往哪里看;
> - **评和注**——从这些地方看出什么来。
>
> 这一阅读方式体现了对文本元素的**辨认与意义生成的同步进行**。"

公式级一句话:
> "阅读能力的核心是阅读方法。阅读方法,即如何阅读,可以概括为,**应该看什么地方和从这些地方应该看出什么来**。"

## 与 Adler-Foster 的对接

**点划评注 ≈ Foster 的"记忆 + 象征 + 模式"三件套**

| Foster 三件套 | 王荣生点划评注 | 做的事 |
|---|---|---|
| 记忆(在哪儿见过类似) | 点、划 | 识别、圈出 |
| 象征(这东西意味着什么) | 评 | 赋予意义 |
| 模式(这是什么类型) | 注 | 关联、类比 |

**点划评注 ≈ Adler 分析阅读的四条规则的操作化**

| Adler 分析阅读(架构性) | 王荣生 | 
|---|---|
| 分类(按体式) | 点(先辨认文本元素) |
| 用最简短的句子说出主题 | 划(找关键语词) |
| 列出全书重要部分 | 评(评论性反思) |
| 找出作者问的问题 | 注(意义生成) |

## 对 stoneyang 项目的核心意义

**王荣生这句话是 stoneyang 文章的中文锚点**,建议在第 3-4 节首段直接引用:

> "王荣生把这件事凝练成两句话 —— '应该看什么地方' + '从这些地方应该看出什么来'。这正是 Foster 2003 年的'记忆 + 象征 + 模式'三件套在中文语境里的对应物。两句话,13 个字,概括了 50 年英美文学教育传统的核心。本文的 AI 流水线要做的第三件事,就是把这 13 个字**工程化** —— 让 Claude 在 ingest 每本书时,自动完成'看什么地方' + '从这些地方看出什么来'的两个动作,并把结果沉淀为 wiki。"

## 与 Karpathy wiki 的结构对应

**点划评注四步 ≈ Karpathy wiki 的 ingest → query → lint 三步的扩展**

- **点 + 划** = ingest 阶段(告诉 LLM 看哪里)
- **评** = query 阶段(让 LLM 从那里生成意义)
- **注** = 双向链接 + _候选 concept(让 LLM 跨书关联)

这意味着 **stoneyang 的 claude.md 事实上已经是王荣生"点划评注"的工程化实现**,只是两人从来没有对话过。
