3. 框架长什么样子？

  【方法论合体】
  - Karpathy LLM wiki(三层架构：raw / wiki / claude.md）
  - MIT 48 小时法（提问协议：5 大框架 / 3 大争议 / 10 道辨别题 + 答错追问）
  - 合体公式：claude.md 把 MIT 协议固化成 Karpathy 的 ingest/lint 规则

  【架构层】
  - raw/                → 14 本书的 PDF（先放阿德勒、宋家博）
  - wiki/index.md       → 总入口
  - wiki/frameworks/    → MIT 第一问产出（5 大共识框架）
  - wiki/debates/       → MIT 第二问产出（3 大争议）
  - wiki/concepts/      → 自然涌现的概念页（双向链接）
  - wiki/tests/         → MIT 第三问产出（辨别题 + 我的答案 + 错题追问）
  - claude.md           → 三层之外，定义 ingest / query / lint 规则

  【7 天交付物】
  1. 一份能跑的 claude.md（Day 1 完成 v0）
  2. raw/ 里 5-7 本最重要的书（阿德勒、宋家博必含）
  3. wiki/ 里 ingest 后的真实状态：
     - 5 张 frameworks/ 页面
     - 3 张 debates/ 页面
     - 1 张 tests/round1.md
     - 10-20 张 concepts/ 页（带双向链接）
     - 1 张 index.md
  4. 一篇 3000 字文章 + 一张 mermaid 知识地图
  5. 1 分钟 demo 视频：演示 ingest 一本新书的完整流水线 + 一次 query → 回写循环

  【显式不交付】
  - 14 本书全 ingest（Day 7 后流水线自然继续跑）
  - 完整的概念图谱（自然增长）
  - 论文式综述（那是另一个产物）
